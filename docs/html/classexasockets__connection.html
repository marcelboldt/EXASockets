Establishes a connection to EXASOL and returns an exasockets_connection object.

Parameters
server The IP address of the EXASOL node to connect to. E.g. '192.168.137.10'
port The Port the EXASOL DB listens on. E.g. 8563
clientName The name of the client, as it will be shown e.g. in the CLIENT column in EXA_(USER/ALL/DBA)_SESSIONS. E.g. the name of the frontend.
username The EXASOL DB user name for authentification. E.g. 'sys'
password The EXASOL user password. E.g. 'exasol'. Transfer is always RSA encrypted.
pwd_len The password length. E.g. 6.
autocommit If the DBMS is supposed to execute a COMMIT automatically after each statement. Default: true (recommended)
use_compression If the result sets are to be transferred compressed. Default: false. Compression is not yet implemented on the client side.
sessionId An ID for the session to be created in EXASOL. Per default a random session ID is created (recommended).
Returns
An exasockets_connection object containing an open connection.

int exasockets_connection::exec_sql ( char *  sql )

Sends an SQL statement to EXASOL and fetches the response.

Sends an SQL stmt and fetches the result set. If a result set handle is received instead of data, then this is returned as an int. Otherwise 0 is returned and the resultSet is stored inside the connection object.

Parameters
sql The SQL statement, e.g. 'select * from schema1.tbl1'
Returns
A result handler to be used with fetch() if available, otherwise 0.
